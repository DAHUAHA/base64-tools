What is a JWT?

What does a JWT look like?
A JWT has three parts separated by dots: header.payload.signature. The header and payload are Base64URL-encoded JSON; the signature is an HMAC of both.
Is it safe to decode JWT in my browser?
Yes — decoding only reads the header and payload (which are plain text, not encrypted). The signature is verified separately. This tool runs entirely in your browser, nothing is sent to any server.
What does "exp" (expiration) mean in the payload?
exp is a Unix timestamp telling when the token expires. This tool automatically shows expiration status and human-readable dates.
Can I forge a JWT with this tool?
You can decode any token, but you cannot forge a valid signature without the secret key. The signature is cryptographically bound to the payload — changing the payload breaks the signature.
What's the difference between HS256, RS256, and other algorithms?
HS256 uses a symmetric secret key (same key to sign and verify). RS256 uses an asymmetric key pair (private key signs, public key verifies). Always use the algorithm your server specifies.
